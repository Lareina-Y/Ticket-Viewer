- backend running locally
- frontend running locally
- database seeded successfully
- `GET /api/tickets/conflicts` working with `bbox`
- optional filters working if implemented
- React page calling the backend
- summary displayed
- ticket results displayed
- loading/error states handled
- map displayed if you implement the bonus section
